NAMING CONVENTIONS
=====================================================================

All files in the Vault follow DDL naming standards.

COUNCIL REVIEWS:
  CR-[PRODUCT]-[NNN]_[TITLE]_[YYYYMMDD].txt
  Example: CR-AUDITFORGE-001_ArchitectureReview_20260313.txt

STANDARDS:
  STD-[DOMAIN]-[NNN]_[TITLE]_v[N].[N].txt
  Example: STD-CORPUS-001_IngestionProtocol_v1.0.txt

PROTOCOLS:
  PRO-[DOMAIN]-[NNN]_[TITLE]_v[N].[N].txt
  Example: PRO-CANONPRESS-001_Pipeline_v1.0.txt

SESSION RECAPS:
  RECAP_[PRODUCT]_[YYYYMMDD]_[SESSION].txt
  Example: RECAP_AuditForge_20260313_PM76.txt

DECISIONS (Architecture Decision Records):
  ADR-[NNN]_[TITLE]_[YYYYMMDD].txt
  Example: ADR-001_SystemOfStructureBoundary_20260313.txt

HANDOFFS:
  HANDOFF_[THREAD-ID]_[PRODUCT]_[YYYYMMDD].txt
  Example: HANDOFF_76_AuditForge_20260314.txt

GENERAL RULE:
  No spaces in filenames. Dashes between words.
  Date format: YYYYMMDD. Always.
  Version format: vN.N (e.g., v1.0, v2.3).
  Prefix always identifies document type.

INGESTION PIPELINE
=====================================================================

Everything in the Vault should have a clear path
into the RAG corpus.

INTAKE FLOW:
  1. Artifact created or collected
  2. Named per convention (above)
  3. Placed in correct Vault folder
  4. Classified: canon or archive
  5. Queued for ingestion
  6. Ingested via dex-ingest.py --build-canon
  7. Verified (chunk count delta)
  8. Logged in ingestion log

CANON vs ARCHIVE:
  Canon (dex_canon collection):
    - Governance documents
    - Ratified standards and protocols
    - Council review verdicts
    - OperatorManifests
    - Build decisions with rationale
    - Anything the operator would want Dex Jr.
      to retrieve with high confidence

  Archive (ddl_archive collection):
    - Raw thread exports
    - Personal message archives
    - Unstructured historical data
    - Draft / pre-ratification content
    - Anything ingested for completeness
      but not governance-critical

NIGHTLY AUTO-INGESTION:
  The corpus already has nightly auto-ingestion.
  The Vault provides the governed intake layer.
  Files placed in the Vault's ingest queue folder
  are picked up automatically.

  Queue folder: C:\Users\dkitc\Downloads\DDL_Ingest

CANON ELEVATION PROTOCOL:
  Not everything that enters the Vault enters as canon.
  Canon elevation requires:
    - Explicit operator designation, OR
    - Council ratification (CR- series), OR
    - Auto-classification by path marker

  Canon content is the highest-confidence source
  for Dex Jr. retrieval. It must be earned, not assumed.

IMMUTABLE RECORD RULE:
  Once a document is ingested as canon, its content
  does not change. If it needs updating, a new version
  is created (v2.0 of the standard, for example).
  The old version is archived, not deleted.
  Dex Jr. can retrieve both. The versioning matters.
