## WHAT THIS PRODUCT IS

blindspot.bet is a data layer for the bettor.
It tracks every bet. It surfaces every pattern. It names the blind spots.

It is NOT a sportsbook.
It is NOT a picks service.
It is NOT gambling advice.
It does NOT predict outcomes.
It does NOT handle real money.
It does NOT claim edge generation — it reveals edge and leak.

Architectural identity: signal -> structure -> edge
The signal is raw bet log data.
The structure is the governed schema — Fact_Bets at center.
The edge is what the splits reveal when the data accumulates.

The house does not have better odds. It has better data.
blindspot.bet is the data layer built for the other side of that equation.

## DATA MODEL

Status: Designed. Not yet built.
Build session pending.

FACT TABLE (center)
  Fact_Bets
    Grain: one row per bet placed
    Fields: date, time, sport, bet_type, platform, odds_entry, stake_tokens,
            result, net_change, token_balance_after, context_note, session_id

  Fact_Rebuys
    Grain: one row per token stack reset
    Fields: date, time, prior_balance, tokens_received, trigger_context

DIMENSIONS
  Dim_Sport         — MLB, NFL, NBA, NCAAMBB, NCAAFB, Soccer, etc.
  Dim_BetType       — Moneyline, Spread, O/U, Prop, Parlay, Live, Futures
  Dim_Platform      — DraftKings, FanDuel, BetMGM, Caesars, etc.
  Dim_MarketContext — macro catalyst, VIX level, news event, bias note
  Dim_User          — user_id, signup_date, token_balance, rebuy_count

BRIDGE / DERIVED
  Parlay legs tracked at row level with parlay_id linking parent bet
  Cash-out logged with exit_value vs full_payout_value as separate fields

PLANNED INTEGRATIONS
  The Odds API      — live lines at bet entry (legal, documented REST API)
  Resend            — waitlist email capture (live)
